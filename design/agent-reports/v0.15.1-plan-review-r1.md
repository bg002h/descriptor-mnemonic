Phase 6 verification likewise.

### M2 — `<paste ABANDON_TPUB_DEPTH3_BIP84 here>` literal placeholders in test code

**Where:** Phase 2 Task 2.3 Step 1 (the `cmd_encode.rs` test snippet).

The plan's test snippet contains literal `<paste...>` placeholders inside
Rust string values. An implementer pasting the snippet verbatim will get
tests calling `parse_key("<paste...")` (base58-decode error). The
`ABANDON_TPUB_DEPTH3_BIP84` const lives under `#[cfg(test)] pub(crate)`
in `parse/keys.rs` and is **not reachable** from
`crates/md-codec/tests/cmd_encode.rs` (separate integration-test
compilation unit).

**Remediation:** add an explicit resolution step before the snippet:
"Substitute the actual tpub string derived in Phase 1 Task 1.1 Step 1
for both `<paste...>` placeholders; run the throwaway derivation test
to recover the value if needed." Or hardcode the literal once Phase 1
runs. Either eliminates the gap.
