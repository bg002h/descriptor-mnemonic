## MEDIUM

### M-1: TLV fingerprints/pubkeys must be sorted by `i` before assignment

`parse_template` (template.rs ~640-651) collects caller-supplied keys/fingerprints in argument
order. `TlvSection::write` (tlv.rs 123-130) requires strictly ascending `idx` and returns
`OverrideOrderViolation`. A user supplying `--fingerprint @1=... --fingerprint @0=...` triggers
this on first encode. Phase 2/3 tests don't catch it because they use ascending or empty TLV.

**Fix:** sort by `i` in `parse_template` before populating `tlv.fingerprints` / `tlv.pubkeys`.

**Status:** Will be fixed in cleanup commit before Phase 6.
