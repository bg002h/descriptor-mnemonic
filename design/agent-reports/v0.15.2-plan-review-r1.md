### I-1: Task 1.3 (`phase-2-low-2`) misses `Testnet4`

`bitcoin = "0.32"` resolves to 0.32.8 here. `Network` has FIVE variants:
`Bitcoin`, `Testnet`, `Testnet4`, `Signet`, `Regtest`.

Reviewer additionally claimed `Network` is `#[non_exhaustive]`; **orchestrator
verified by reading `~/.cargo/registry/.../bitcoin-0.32.8/src/network.rs`
lines 70-88: `Network` is NOT `#[non_exhaustive]`** (the `#[non_exhaustive]`
attributes at lines 258 and 297 belong to other types). So exhaustive
matching IS allowed and IS the correct fix for the original goal ("future
variants become a compile error"). The reviewer's wildcard recommendation
isn't required, but `Testnet4` MUST be in the arm.

**Fix applied to Task 1.3:** include `Testnet4` in the named arm, route
to the same testnet xpub-version branch (BIP 32 testnet bytes cover all
testnet flavors per spec).
