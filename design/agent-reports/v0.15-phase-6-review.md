## HIGH

### H-1: `force_chunked` was silently ignored in vectors generator (FIXED post-review)

The manifest defines `wsh_multi_chunked` with `force_chunked: true` but `cmd/vectors.rs` always
called `encode_md1_string` regardless. The committed `.phrase.txt` was a single-phrase encoding,
not chunked. The `force_chunked` field was effectively dead. Phase 8's corpus diff would have
pinned the wrong golden value.

**Fix applied:** branch on `v.force_chunked`; when true, use `split` + `derive_chunk_set_id` and
write a multi-line `chunk-set-id: 0x...\n<chunks>` payload. Corpus regenerated.

## MEDIUM

### M-1: Plan's `_ctx` binding silently dropped (benign)

Plan code had `let _ctx = ctx_for_template(v.template);` (intent: surface context explicitly).
Implementation drops it because `parse_template` derives ctx internally. No functional impact.
