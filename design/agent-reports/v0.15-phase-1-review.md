## IMPORTANT

### I-1: `n` overflow → panic on `@255`

**File:** `crates/md-codec/src/bin/md/parse/template.rs` line ~143 (`resolve_placeholders`)
**Confidence: 85**

```rust
let n = (by_i.keys().max().copied().ok_or_else(...)? as usize + 1) as u8;
```

If the highest `@i` index is 255, `255usize + 1 = 256`, `256 as u8 = 0`. With `n = 0`, the dense
check loop runs zero iterations. The next line `let at0 = by_i[&0]` panics if `@0` is absent.

Suggested fix:
```rust
let max_i = by_i.keys().max().copied().ok_or_else(|| CliError::TemplateParse("no placeholders".into()))?;
if max_i >= 32 { return Err(CliError::TemplateParse(format!("@{max_i} exceeds max key index 31"))); }
let n = max_i + 1;
```

**Status:** Unreachable in normal use (32-key wire limit). Will be tracked as a follow-up; not
blocking Phase 2.
