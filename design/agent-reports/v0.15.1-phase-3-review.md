## LOW (defer to FOLLOWUPS at Phase 6 under `v0.15.2`)

### `v0.15.1-phase-3-low-1` — Latent fragility if clap's ArgGroup ever fails to block zero-arg invocation

`build_descriptor` phrase path would call `reassemble(&[])` (exit 1) if
clap ever delivers `phrases = []` with no template. `ArgGroup::required(true)`
on a positional `num_args = 0..` should prevent this, and the
`address_no_input_exits_2` test guards against it. Not a current bug; just
a latent fragility if clap behavior changes. Confidence: 40.

**Suggested defensive guard (deferred):** at the top of `build_descriptor`,
add `if args.phrases.is_empty() && args.template.is_none() { return Err(BadArg("...")); }`.
